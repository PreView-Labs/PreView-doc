본 문서는 **PreView AI** 프로젝트의 기능 우선순위(MoSCoW)를 상세히 설명합니다. 한정된 개발 기간 내에 핵심 가치를 전달하기 위해 개발 범위를 명확히 정의합니다.

* **작성자**: [고동현](https://github.com/rhehdgus8831)
* **작성일**: 2025-12-02
* **문서 버전**: v1.0.0

**대상 독자:**

* **기획자 / PM**: 프로젝트의 단계별 마일스톤과 서비스 로드맵을 수립하는 담당자
* **백엔드 개발자**: API 개발 순서와 데이터베이스 설계 범위를 결정하는 개발자
* **프론트엔드 개발자**: 화면 구현의 우선순위를 정하고 핵심 UX에 집중하는 개발자
* **AI 엔지니어**: 프롬프트 엔지니어링의 고도화 단계(텍스트 → 음성 → 멀티모달)를 파악하는 담당자
* **DevOps 엔지니어**: 배포 환경(SSE, HTTPS) 구축 시점을 조율하는 담당자

## 기능 우선순위 (MoSCoW)

PreView AI 프로젝트의 MVP 구현을 위한 핵심 기능과 향후 확장 계획은 다음과 같습니다.

### 1. Must Have (MVP - 필수 구현 기능)

현재 프로젝트 기간 내에 **반드시 구현하여 배포해야 할 핵심 기능**입니다.

* **간편 로그인**:
    * 카카오톡 OAuth2 연동을 통한 원클릭 회원가입 및 로그인.
    * JWT(Access/Refresh Token) 발급 및 보안 쿠키(HttpOnly) 처리.
* **이력서 관리**:
    * PDF 파일 업로드 및 텍스트 자동 추출(Parsing).
    * 추출된 텍스트 전처리(불용어 제거, 개인정보 마스킹 등) 후 DB 저장.
* **AI 면접관 (Core)**:
    * **Google Gemini API** 연동 (Free Tier 활용).
    * **페르소나 시스템**: 시스템 프롬프트를 통해 '10년 차 시니어 개발자' 인격 부여.
    * **질문 생성**: 이력서 내용을 기반으로 한 맞춤형 초기 질문 생성.
* **실시간 채팅 면접**:
    * **SSE(Server-Sent Events)** 기술을 적용하여 AI의 답변이 실시간으로 타자 치듯 출력되는 효과 구현.
    * **꼬리 질문(Deep Dive)**: 사용자의 답변이 입력되면, 내용을 분석하여 논리적 허점을 파고드는 심층 질문 생성.
    * 면접 종료 기능 (사용자 요청 시 즉시 중단 및 결과 분석).
* **피드백 리포트**:
    * 면접 종료 직후 종합 점수(100점 만점) 산출.
    * 긍정적 평가(Good)와 보완할 점(Bad) 요약.
    * 각 문답별 상세 피드백(모범 답안 제안) 제공.

### 2. Should Have (차기 업데이트 - 우선적으로 고려할 기능)

MVP가 안정적으로 동작한 후 **품질 향상을 위해 추가할 기능**입니다.

* **이력서 보관함**:
    * 매번 업로드할 필요 없이, 최근 업로드한 이력서를 저장하고 불러오는 기능.
* **면접관 성향 선택 (UI)**:
    * '압박 면접형', '인성 면접형', 'CS 지식형' 등 면접관 스타일을 사용자가 직접 선택하는 옵션 제공.
* **음성 인식 (STT) & 합성 (TTS)**:
    * (Chrome Web Speech API 활용) 사용자가 마이크로 답변하면 텍스트로 변환.
    * AI의 질문을 음성(Voice)으로 읽어주는 기능 추가하여 실전성 강화.
* **마이페이지 대시보드**:
    * 과거 면접 기록 리스트 조회 (무한 스크롤).
    * 면접 점수 변화 추이 그래프(Chart.js) 시각화.

### 3. Could Have (중장기 확장 - 장기적으로 고려할 기능)

향후 프로젝트 규모가 커지거나 유료화 모델 도입 시 **추가할 수 있는 아이디어**입니다.

* **화상 면접 (WebRTC)**:
    * 카메라를 켜고 면접을 진행하며, 사용자의 시선 처리나 표정 변화를 분석.
* **기업별 맞춤 모드**:
    * "Naver 모드", "Toss 모드" 등 특정 기업의 기출문제 데이터를 학습한 특화 면접관.
* **커뮤니티 & 공유**:
    * 자신의 면접 결과 리포트를 익명으로 커뮤니티에 공유하여 다른 사용자에게 피드백 요청.
    * "이달의 베스트 답변" 랭킹 시스템.
* **모바일 앱 출시**:
    * React Native 또는 Flutter를 활용한 네이티브 앱 확장.

### 4. Won’t Have (초기 제외 - 현재 고려하지 않는 기능)

프로젝트의 복잡도를 낮추기 위해 **현재 단계에서는 구현하지 않는 기능**입니다.

* **회원가입 폼 (일반 가입)**: 아이디/비밀번호 찾기 등 관리 소요가 커서 소셜 로그인으로만 제한.
* **결제 시스템**: PG사 연동 및 유료 구독 모델 (현재는 포트폴리오 목적이므로 전면 무료).
* **사람 멘토링 매칭**: 현직자와 사용자를 연결해 주는 O2O 기능.
* **다국어 지원**: 영어 면접 기능은 MVP 범위에서 제외 (한국어 집중).
