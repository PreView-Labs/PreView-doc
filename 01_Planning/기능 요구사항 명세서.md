## 1\. 개요 (Overview)

PreView AI는 **이력서 기반 맞춤형 AI 모의 면접 플랫폼**입니다. 사용자의 이력서를 분석하여 실제 면접관처럼 꼬리 질문을 던지고, 객관적인 데이터 기반의 피드백을 제공하여 취업 준비생의 실전 감각을 높이는 것을 목표로 합니다.

-----

## 2\. 목적 (Purpose)

* **초개인화**: 이력서 내용을 분석하여 사용자에게 딱 맞는 질문을 생성합니다.
* **실전 감각 배양**: 꼬리 질문 알고리즘을 통해 압박 면접 상황을 대비합니다.
* **객관적 평가**: 모호한 피드백 대신 점수와 키워드 분석을 통해 개선점을 명확히 제시합니다.
* **접근성 향상**: 별도의 설치 없이 웹 브라우저에서 즉시 모의 면접을 진행합니다.

-----

## 3\. 핵심 기능 (Core Features)

* **간편 로그인**: 카카오 OAuth2를 통한 3초 회원가입/로그인.
* **이력서 파싱**: PDF 파일을 업로드하면 텍스트를 추출하고 핵심 역량을 분석.
* **AI 실시간 면접**: Google Gemini API와 SSE(Server-Sent Events)를 활용한 실시간 스트리밍 대화.
* **페르소나 설정**: '친절한 면접관', '압박 면접관' 등 성향 선택 가능.
* **상세 피드백**: 면접 종료 후 종합 점수, 잘한 점, 보완할 점 리포트 제공.

## 4\. 기능 요구사항 (Functional Requirements)

### 4.1 회원/인증 (Auth)

| ID | 기능 상세 | 우선순위 | 비고 |
| :--- | :--- | :--- | :--- |
| FR-01 | 카카오톡 SSO 로그인 | **MUST** | OAuth 2.0 기반 소셜 로그인 (인가코드/토큰 처리) |
| FR-02 | JWT 토큰 발급 및 관리 | **MUST** | Access(30분)/Refresh(2주) 토큰 전략, HttpOnly 쿠키 사용 |
| FR-03 | 자동 로그인 (토큰 갱신) | **MUST** | Access 토큰 만료 시 Refresh 토큰으로 자동 재발급 |
| FR-04 | 로그아웃 | **MUST** | 쿠키 삭제 및 DB 내 Refresh 토큰 만료 처리 |
| FR-05 | 마이페이지 (히스토리) | **MUST** | 과거 면접 기록 리스트 조회 및 상세 리포트 다시 보기 |

* **회원가입/로그인**: 별도의 회원가입 폼 없이 카카오 계정만으로 진입 장벽 최소화.
* **보안**: JWT를 활용한 Stateless 인증 방식 적용.

### 4.2 이력서 관리 (Resume)

| ID | 기능 상세 | 우선순위 | 비고 |
| :--- | :--- | :--- | :--- |
| FR-06 | PDF 파일 업로드 | **MUST** | 최대 10MB 제한, PDF 포맷만 허용 |
| FR-07 | 텍스트 추출 (Parsing) | **MUST** | Apache PDFBox 등을 활용하여 텍스트 데이터 추출 |
| FR-08 | 핵심 키워드 분석 | **MUST** | 추출된 텍스트를 AI에게 전송하여 기술 스택/프로젝트 경험 요약 |
| FR-09 | 이력서 저장 및 관리 | SHOULD | 한 번 업로드한 이력서를 재사용할 수 있도록 저장 |

* **파싱 로직**: PDF 내의 텍스트를 줄바꿈 기준으로 명확하게 추출하여 프롬프트에 주입할 수 있도록 전처리 수행.

### 4.3 모의 면접 진행 (Interview)

| ID | 기능 상세 | 우선순위 | 비고 |
| :--- | :--- | :--- | :--- |
| FR-10 | 면접 설정 (Setup) | **MUST** | 이력서 선택, 지원 직무(BE/FE/AI), 면접관 성향 선택 |
| FR-11 | 질문 생성 (Generation) | **MUST** | 이력서 내용 기반 초기 질문 1개 생성 |
| FR-12 | 채팅 인터페이스 | **MUST** | 사용자 답변 입력창, 말풍선 UI, 스크롤 자동 이동 |
| FR-13 | 실시간 답변 스트리밍 | **MUST** | **SSE(Server-Sent Events)** 적용하여 AI 답변을 타자 치듯 출력 |
| FR-14 | 꼬리 질문 로직 | **MUST** | 사용자 답변이 짧거나 불명확할 경우 심층 질문 생성 |
| FR-15 | 면접 종료 | **MUST** | 사용자가 종료 버튼 클릭 시 대화 중단 및 분석 요청 |

* **스트리밍(SSE)**: WebSocket 대신 단방향 통신인 SSE를 사용하여 구현 복잡도를 낮추고 AI 응답 체감 속도를 높임.
* **프롬프트**: 시스템 프롬프트에 "너는 10년 차 시니어 개발자 면접관이다"라는 페르소나 주입.

### 4.4 결과 분석 및 피드백 (Feedback)

| ID | 기능 상세 | 우선순위 | 비고 |
| :--- | :--- | :--- | :--- |
| FR-16 | 종합 점수 산출 | **MUST** | 논리성, 직무 적합성, 태도 등을 100점 만점으로 환산 |
| FR-17 | 긍정/부정 피드백 요약 | **MUST** | "두괄식 답변이 좋습니다", "CS 지식이 부족합니다" 등 요약 |
| FR-18 | 문답별 상세 코멘트 | **MUST** | 각 질문-답변 쌍에 대해 AI가 "모범 답안" 또는 "개선 팁" 제공 |
| FR-19 | 결과 리포트 저장 | **MUST** | DB에 JSON 형태로 분석 결과 저장 (추후 조회용) |

* **평가 기준**: AI에게 명확한 채점 기준표(Rubric)를 프롬프트로 제공하여 일관성 있는 점수 산출.

## 5\. 제약 사항 (Constraints)

* **비즈니스 제약**

    * **무료 LLM 사용**: 비용 절감을 위해 Google Gemini API (Free Tier)를 사용하므로, 분당 요청 횟수(RPM) 제한이 있을 수 있음.
    * **PDF 전용**: 워드(doc), 한글(hwp) 파일은 MVP 단계에서 지원하지 않음.
    * **텍스트 전용**: 음성(Voice) 면접 기능은 MVP 이후 고도화 단계에서 지원.

* **기술적 제약**

    * **브라우저 호환성**: Chrome, Safari, Edge 등 모던 브라우저 최신 버전 지원.
    * **모바일 지원**: 반응형 웹(Responsive Web)으로 구현하여 모바일에서도 이용 가능해야 함.

## 6\. 용어 정의 (Glossary)

* **SSE (Server-Sent Events)**: 서버에서 클라이언트로 데이터를 실시간 스트리밍하는 단방향 통신 기술. AI의 답변 생성 과정을 실시간으로 보여주기 위해 사용.
* **꼬리 질문 (Follow-up Question)**: 지원자의 답변 내용을 바탕으로 더 깊이 파고드는 추가 질문.
* **프롬프트 엔지니어링 (Prompt Engineering)**: LLM(AI)이 원하는 결과물을 내놓도록 지시어(Prompt)를 최적화하는 과정.
* **파싱 (Parsing)**: PDF 파일 같은 비정형 데이터를 컴퓨터가 이해할 수 있는 텍스트 데이터로 변환하는 과정.
* **페르소나 (Persona)**: AI에게 특정 인격(예: 까칠한 면접관)을 부여하여 일관된 톤앤매너를 유지하게 하는 것.
