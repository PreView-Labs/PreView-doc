# PreView AI 비기능 요구사항 명세서

본 문서는 **PreView AI** 서비스의 비기능 요구사항(Non-functional Requirements, NFR)을 정의하여 시스템의 품질 속성, 제약 조건, 성능 목표를 명확히 합니다.

* **작성자**: [고동현](https://github.com/rhehdgus8831)
* **작성일**: 2025-12-02
* **문서 버전**: v1.0.0
* **기반 문서**: PreView AI 프로젝트 개요, 사용자 흐름도, 기능 우선 순위

**대상 독자:**

* **기획자 / PM**: 서비스 품질 기준(성능, 보안)을 이해하고 기획 방향성을 검토하는 담당자.
* **백엔드 개발자**: API 응답 속도 및 SSE 스트리밍 안정성을 확보하기 위한 기술적 목표를 파악하는 담당자.
* **프론트엔드 개발자**: 사용자 인터랙션 반응 속도 및 웹 접근성을 고려하여 UI를 구현하는 담당자.
* **DevOps 엔지니어**: 서버 가용성, 배포 자동화 성공률, 보안 설정(HTTPS) 기준을 준수하는 담당자.
* **QA / 테스트 엔지니어**: 비기능 요구사항에 기반한 성능 테스트(Load Test) 및 보안 점검 시나리오를 작성하는 담당자.

---

## 1. 개요

PreView AI 프로젝트는 실시간 대화형 AI 면접 서비스로, 사용자가 실제 면접과 유사한 경험을 할 수 있도록 **빠른 응답 속도(Latency)**와 **높은 가용성(Availability)**이 핵심 성공 요소입니다. 또한, 사용자의 민감한 정보(이력서)를 다루므로 **철저한 보안(Security)**이 요구됩니다.

---

## 2. 비기능 요구사항 상세 명세

요구사항은 **'NFR-(분류)-(번호)'** 형식으로 ID를 부여하며, 주요 분류는 성능, 보안, 가용성, 유지보수성, 사용성/접근성으로 구분합니다.

### 2.1 성능 (Performance)

| ID | 요구사항 정의 | 근거 / 관련 기능 | 측정 지표 | 목표 값 | 우선 순위 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **NFR-PER-01** | AI 답변 스트리밍 시작 속도 | 사용자 질문 입력 후 첫 번째 글자가 출력되기까지의 시간 (TTFB) | Time To First Byte | **2초 이내** | Must Have |
| **NFR-PER-02** | 챗봇 응답 지연 시간 | 전체 답변 생성이 완료되기까지의 시간 (사용자 체감 대기 시간 최소화) | Total Response Time | **15초 이내** | Must Have |
| **NFR-PER-03** | 이력서 파싱 속도 | PDF 업로드 직후 텍스트 추출 및 분석 완료까지 소요되는 시간 | Processing Time | **5초 이내** | Must Have |
| **NFR-PER-04** | 동시 접속자 처리 | 모의 면접 동시 진행 가능 인원 (t3.small 기준) | Concurrent Users | **50명 이상** | Must Have |
| **NFR-PER-05** | API 평균 응답 속도 | 일반적인 CRUD(조회, 저장) API의 응답 속도 | Average Latency | **500ms 이내** | Must Have |

| :--- | :--- | :--- | :--- | :--- | :--- |
| **NFR-SEC-01** | 통신 암호화 | 모든 데이터 전송 구간(프론트↔백엔드)에 **HTTPS(TLS 1.2+)** 적용 | SSL Labs 등급 | **A 등급** | Must Have |

## 3. 제약 사항 (Constraints)

* **비즈니스 제약:**
    * AI 모델은 비용 문제로 인해 **Google Gemini API (Free Tier)**를 우선적으로 사용하며, 요청 량 제한(Rate Limit)을 고려해야 합니다.
    * 클라우드 인프라 비용은 **월 2만 원 이하** (AWS Free Tier 적극 활용)로 유지해야 합니다.
* **기술적 제약:**
    * 백엔드 프레임워크는 **Java Spring Boot 3.x** 버전을 사용합니다.
    * 프론트엔드는 **React (Vite)** 기반으로 개발합니다.
    * 배포 환경은 **Docker** 컨테이너 기반으로 표준화합니다.
* **법적 제약:**
    * 사용자 이력서 데이터는 개인정보보호법에 의거하여, 사용자가 삭제를 요청하거나 탈퇴 시 **즉시 파기**해야 합니다.

## 4. 용어 정의 (Glossary)

| 용어 | 정의 |
| :--- | :--- |
| **TTFB (Time To First Byte)** | 서버에 요청을 보낸 후 첫 번째 바이트(데이터)가 도착하기까지 걸리는 시간. AI 답변의 반응 속도를 측정하는 핵심 지표. |
| **SSE (Server-Sent Events)** | 서버에서 클라이언트로 데이터를 실시간으로 스트리밍하는 단방향 통신 기술. |
| **JWT (JSON Web Token)** | 유저를 인증하고 식별하기 위한 토큰 기반 인증 방식. |
| **SLA (Service Level Agreement)** | 서비스 제공자가 보장하는 서비스 가동률 등의 품질 수준. |
| **HttpOnly Cookie** | 자바스크립트로 접근할 수 없는 쿠키로, XSS(크로스 사이트 스크립팅) 공격을 방지하기 위해 사용. |
